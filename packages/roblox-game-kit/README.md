# Roblox GameKit

Reusable Roblox modules distilled from EggBreakers, GroanTubeHero, and the
RobloxAIDev Prop Hunt surface.

The package is intentionally source-first:

- `module-catalog.json` names each reusable module family and the source games
  that motivated it.
- `inventory/source-library-inventory.json` is generated by
  `node scripts/inventory_reusable_game_libraries.mjs`.
- `src/ReplicatedStorage/GameKit` contains shared modules.
- `src/ServerScriptService/GameKit` contains server-only modules.

Future games can copy or Rojo-map this package, then add project adapters for
game-specific config, art, datastore names, and UI rendering.

## Cleanup Plan

Behavior lock:

- Keep original game repos unchanged.
- Inventory every scanned source library and assign it to a module family.
- Test that catalog families, module files, and generated inventory stay aligned.

Passes:

1. Extract small deep modules for repeated behavior instead of shallow wrappers.
2. Keep game-specific data in adapters/config registries.
3. Preserve grounded compatibility behavior, but make it explicit at interfaces.
4. Leave large source-project rewrites for later migration branches.

Fallback findings:

- Studio-only DataStore behavior is a grounded compatibility fallback.
- Debug visual fallbacks are valid only when release gates reject them.
- Procedural visible affordances must carry explicit release approval metadata.
- Silent rescue or broad `pcall` paths should be moved behind policy modules
  before being reused in new games.
